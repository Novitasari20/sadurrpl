Hasil Saduran Saya Hariini :
  1. Pada pertemuan ini kita akan membahas Sequence Diagram
  2. Sequence Diagram adalah suatu diagram yang menjelaskan interaksi objek dan menunjukkan (memberi tanda atau petunjuk) komunikasi diantara objek-objek tersebut.
  3. Sequence diagram digunakan untuk menjelaskan perilaku pada sebuah skenario dan menggambarkan bagaimana entitas dan sistem berinteraksi,
    termasuk pesan yang dipakai saat interaksi.Semua pesan digambbarkan dalam urutan pada eksekusi.
  4. Sequence Diagram menggambarkan urutan proses yang terjadi pada sebuah halaman/form pada aplikasi/sistem
  5. Sequence Diagram sangat erat hubungannya dengan Deskripsi UseCase + Rancangan Layar + Class Diagram
  6. Elemen inti pada Sequence Diagram :
    - Lifeline (Actor), Lifeline (Boundary), Lifeline (Control), Lifeline (Entity), Message
    - Actor => pengguna aplikasi/sistem
    - Boundary => form/halaman di aplikasi
    - Control => code program
    - Entity => tabel
    - Message => aktivitas di aplikasi/sistem
  7. Actor berinteraksi dengan Boundary, Boundary berinteraksi dengan Control, Control berinteraksi dengan Entity
    -Masing-masing interaksi digambarkan dengan garis panah berisi aktivitas (message)
  8. Untuk pertemuan ini kita akan membuat sequence diagram menggunakan asta community,
    berdasarkan proses bisnis yang sudah kita pelajari pada pertemuan-pertemuna sebelumnya.
  9. Untuk membuat Sequence Diagram caranya yaitu :
    - Buka aplikasi astah community
    - Buat file baru,
    - Lalu beri nama Sequence Diagram
    - Kemudian buat Sequence Diagram berdasarkan analisa proses bisnis yang sudah dijelaskan pada pertemuan sebelumnya.
    - Terakhir, simpan file Sequnce Diagramnya.
  10. Tambahkan file pendukung seperti Sequence Diagram yang telah dibuat tadi ke github.com
    caranya yaitu : Difolder 7 klik add file, kemudian upload file, lalu Drag file astahnya , dan terakhir di Commit.
  11. Terakhir, kirim link hasil saduran kita ke dosen pengampu matakuliah
